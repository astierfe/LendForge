## üêõ Bug Fixes

### ANO_013: Bot Borrowed Amount Conversion (Critical)
- **Problem:** Bot displayed borrowed amount as $1.124 billion instead of $366
- **Root Cause:** Treated borrowed Wei ETH (18 decimals) as USD (8 decimals)
- **Fix:** Added `convert_borrowed_to_usd()` method in `web3_client.py`
- **Files:** `bot/src/clients/web3_client.py`, `position_monitor.py`, `profit_calculator.py`, `liquidation_check.py`
- **Impact:** Bot now correctly calculates liquidation profitability and debt amounts

### ANO_008: Liquidation Missing Collateral Transfer (Critical)
- **Problem:** `LendingPool.liquidate()` accepts payment but never transfers seized collateral to liquidator
- **Security Impact:** Liquidators lose 100% of funds, protocol accumulates bad debt
- **Temporary Fix:** Manual script `transfer_liquidated_collateral.sh` (testing only)
- **Files:** `scripts/transfer_liquidated_collateral.sh`, `bot/src/services/liquidator.py` (warning message)
- **Documentation:** `_docs/issues/ANO_008_liquidation-missing-collateral-transfer.md`
- **Status:** Contract fix required for production (add `CollateralManager.seizeCollateral()`)

### Frontend LTV Configuration
- **Fix:** Corrected USDC/DAI LTV ratios from 75% to 90% in `frontend/lib/contracts/config.ts`
- **Impact:** Aligns frontend with contract configuration (90% per CollateralManager)

- **Problem:** Frontend showed stale values from subgraph, updated only after transactions

- `UserPositionCard.tsx` - Real-time borrowed amount (no more stale debt values)

**Benefits:**

- Consistent with `OraclePricesCard` pattern
